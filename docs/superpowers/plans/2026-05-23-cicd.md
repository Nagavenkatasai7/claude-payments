# CI/CD Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace manual `vercel --prod` with a GitHub-backed CI/CD pipeline: PRs run typecheck/lint/test/build, merges to `main` auto-deploy to Vercel, branch protection enforces the workflow, and a Playwright smoke test guards the live site.

**Architecture:** GitHub repo as source of truth → GitHub Actions for checks → Vercel native GitHub integration for deploys → branch protection on `main` to make the checks mean something. A single CI workflow on PR + push, plus a Playwright job that runs after production deploys.

**Tech Stack:** GitHub Actions (`actions/checkout@v4`, `actions/setup-node@v4`), Node 24, npm, ESLint with `eslint-config-next`, TypeScript `tsc --noEmit`, Vitest, Next 16 `next build`, Vercel GitHub App, Playwright with Chromium.

**Spec:** [docs/superpowers/specs/2026-05-23-cicd-design.md](../specs/2026-05-23-cicd-design.md)

## Task 3: Add the CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Write the workflow file**

```yaml
name: CI

on:
  pull_request:
  push:
    branches: [main]

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  ci:
    name: ci
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '24'
          cache: 'npm'

      - name: Install
        run: npm ci

      - name: Typecheck
        run: npm run typecheck

      - name: Lint
        run: npm run lint

      - name: Test
        run: npm test

      - name: Build
        run: npm run build
```

- [ ] **Step 2: Commit to the same `chore/add-lint-typecheck` branch**


## Task 5: Branch protection on `main`

**Files:**
- No code changes; one `gh api` call.

- [ ] **Step 1: Apply the protection rule**

```bash
gh api -X PUT repos/Nagavenkatasai7/claude-payments/branches/main/protection \
  -H "Accept: application/vnd.github+json" \
  -f required_status_checks[strict]=true \
  -f required_status_checks[contexts][]='ci / ci' \
  -F enforce_admins=true \
  -F required_pull_request_reviews[required_approving_review_count]=0 \
  -F required_pull_request_reviews[dismiss_stale_reviews]=true \
  -F restrictions= \
  -F allow_force_pushes=false \
  -F allow_deletions=false
```

(`-f` is string field, `-F` is bool/number/null/array. The `restrictions=` empty value disables push-actor restrictions.)

- [ ] **Step 2: Verify negative case — direct push to main is rejected**

```bash
git checkout main
git commit --allow-empty -m "test: verify branch protection rejects direct push"
git push origin main 2>&1 | tee /tmp/push.log
grep -q "protected branch" /tmp/push.log && echo OK || echo BROKEN
```

Expected: push is rejected (`protected branch hook declined`). Undo the local commit:

```bash
git reset --hard HEAD~1
```

- [ ] **Step 3: Verify positive case — PR with green CI can merge**

(Already proven in Task 3 step 5, but re-verify on a no-op PR.)

```ts
  testDir: './tests/e2e',
  timeout: 30_000,

  retries: process.env.CI ? 1 : 0,
  reporter: process.env.CI ? 'github' : 'list',
```

- [ ] **Step 4: Write the failing test**

Create `tests/e2e/dashboard-smoke.spec.ts`:

```ts
import { test, expect } from '@playwright/test';

const USERNAME = process.env.E2E_USERNAME ?? 'forextransfer';
const PASSWORD = process.env.E2E_PASSWORD ?? 'forex@123';

test('staff can log in and reach dashboard pages', async ({ page }) => {
  await page.goto('/login');
  await page.getByLabel(/username/i).fill(USERNAME);
  await page.getByLabel(/password/i).fill(PASSWORD);
  await page.getByRole('button', { name: /sign in/i }).click();

  await expect(page).toHaveURL(/\/dashboard/);
  await expect(page.getByText(/overview/i).first()).toBeVisible();

  await page.getByRole('link', { name: /analytics/i }).click();
  await expect(page).toHaveURL(/\/dashboard\/analytics/);
  // Recharts renders an SVG; existence is enough.
  await expect(page.locator('svg').first()).toBeVisible();

  await page.getByRole('link', { name: /transactions/i }).click();
  await expect(page).toHaveURL(/\/dashboard\/transactions/);
  await expect(page.getByRole('table').or(page.getByText(/no transfers/i))).toBeVisible();
});
```

- [ ] **Step 5: Run locally against production to confirm it passes**


Expected: 1 passed. If selectors don't match the actual login form, adjust
to match the rendered labels (the form uses `username`/`password` fields).

- [ ] **Step 6: Create `.github/workflows/smoke.yml`**

```yaml
name: Smoke

on:
  deployment_status:

jobs:
  smoke:
    if: github.event.deployment_status.state == 'success' && github.event.deployment.environment == 'Production'
    runs-on: ubuntu-latest
    timeout-minutes: 5

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '24'
          cache: 'npm'

      - run: npm ci

      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium

      - name: Smoke test
        env:
          BASE_URL: ${{ github.event.deployment_status.target_url }}
          E2E_USERNAME: ${{ secrets.E2E_USERNAME }}
          E2E_PASSWORD: ${{ secrets.E2E_PASSWORD }}
        run: npm run e2e

      - name: Upload trace on failure
        if: failure()
        uses: actions/upload-artifact@v4
        with:
          name: playwright-trace
          path: test-results/
          retention-days: 7
```
